An easy way to start using l10n in your project

You have project that is already working and need to localize it to different languages. Don't do that boring copy&pasting from your HTML templates to newly created message resource - let angular-l10n do that for you.

Just add l10n directive (like this string is localized already) to every element you need to localize. For example you had:

<p>This text should be localized</p>

After you will have

<p l10n-text="myApp.shouldBeLocalized">This text should be localized</p>

Ok, now add <script src="l10n-export.js"></script> to your page.

Put following code somewhere in your app or setup it as separate route.

<textarea ng-controller="L10nExportCtrl" rows="20" class="container span12">
angular.module('l10n-{{ currentLocale }}', ['l10n']).config(['l10nProvider', function(l10n){
	l10n.add('{{ currentLocale }}', {{ resources }});
}]);
</textarea>

Help angular-l10n to know more about your message resources. You will need to open each template in a browser. Hope later I will be able to automate this in some way.

After this you need to get data out from exporter. Open the page with controller you had setup two steps ago and you will see all the message resources that it found in your templates. Copy it and paste into you message resources
